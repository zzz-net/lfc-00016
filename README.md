# 巡逻棋 Patrol Chess

一款纯本地可玩的回合制巡逻棋小游戏。玩家在 8×8 网格中规划巡逻路径，通过捕获事件点获得分数，同时避开障碍物和危险事件。支持存档、读档、撤销操作、日志回放和存档包导入导出功能。

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

## 游戏规则

### 1. 游戏目标

在 50 回合内达到 100 分获胜。

### 2. 基本操作

#### 移动方式

- **鼠标点击**：点击玩家相邻的格子进行移动
- **键盘控制**：
  - `W` / `↑` 向上移动
  - `S` / `↓` 向下移动
  - `A` / `←` 向左移动
  - `D` / `→` 向右移动
- **快捷键**：
  - `Ctrl + Z` 撤销上一回合
  - `Ctrl + N` 开始新游戏

### 3. 游戏元素

| 元素 | 图标 | 说明 |
|--------|------|------|
| 巡逻单位 | ● 绿色发光圆点 | 玩家控制的单位，从左上角 (0,0) 开始 |
| 普通事件 | ◆ 橙色 | 捕获后 +10 分 |
| 奖励事件 | ★ 金色 | 捕获后 +30 分 |
| 危险事件 | ☠ 红色 | 捕获后 -20 分，**游戏立即结束** |
| 障碍物 | ▓ 灰色方块 | 不可通过 |

### 4. 移动规则

- 玩家每回合只能向**上下左右**四个方向移动 **1 格
- **不能移动到棋盘外**（8×8 边界）
- **不能穿过障碍物**
- **非法移动时**：
  - 系统提示"非法移动"
  - 回合数**不变化**
  - 分数**不变化**
  - 事件位置**不变化**
  - 游戏状态完全不变

### 5. 事件系统

#### 事件捕获

- 玩家移动到事件所在格子即捕获该事件
- 事件捕获后立即从棋盘上消失
- 分数立即结算

#### 系统回合

每次玩家合法移动后，系统会执行以下操作之一：

- **50% 概率**：随机移动一个现有事件 1 格
- **30% 概率**：生成一个新事件
  - 70% 普通事件 (+10分)
  - 15% 奖励事件 (+30分)
  - 15% 危险事件 (-20分)
- **20% 概率**：无变化

### 6. 失败条件

满足以下任一条件，游戏立即结束：

1. **捕获危险事件**：移动到红色危险事件格子
2. **分数为负**：分数低于 0 分
3. **回合耗尽**：50 回合结束时分数未达到 100 分

### 7. 胜利条件

- 分数达到或超过 100 分，游戏胜利

### 8. 撤销功能

- **撤销上一回合**：恢复到上一回合开始前的状态
- 撤销后恢复：
  - 玩家位置
  - 所有事件位置
  - 分数
  - 回合数
  - 回放日志
- 可多次撤销，直到游戏初始状态
- 回放模式下无法撤销

### 9. 存档系统

#### 手动存档

- 5 个存档槽位
- 可自定义存档名称
- 存档内容包括：
  - 当前棋盘状态
  - 玩家位置
  - 所有事件位置
  - 分数和回合数
  - 完整历史记录
  - 回放日志

#### 自动存档

- 每次合法移动后自动保存
- 关闭页面后重新打开可读取自动存档
- 自动存档独立于手动存档

#### 读取存档

- 读取存档后可继续游戏
- 读取存档后可进行回放

### 10. 回放功能

- 查看完整的游戏过程
- 支持：
  - 逐帧前进/后退
  - 自动播放（0.5x / 1x / 2x / 4x 速度)
  - 拖动进度条跳转到指定回合
  - 退出回放回到当前状态

### 11. 操作日志

记录每回合的详细操作：

- 移动记录（方向、起始位置、目标位置)
- 事件捕获（事件类型、分数变化)
- 系统操作（事件移动、新事件生成)
- 非法移动提示
- 游戏结束原因

## 技术栈

- **框架**: React 18 + TypeScript
- **构建**: Vite 5
- **样式**: TailwindCSS 3
- **状态管理**: Zustand
- **图标**: Lucide React
- **存储**: localStorage
