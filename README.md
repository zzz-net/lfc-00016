# 巡逻棋 Patrol Chess

一款纯本地可玩的回合制巡逻棋小游戏。玩家在 8×8 网格中规划巡逻路径，通过捕获事件点获得分数，同时避开障碍物和危险事件。支持存档、读档、撤销操作、日志回放、存档包导入导出，以及**关卡工坊**——玩家可以手动设计自定义关卡、导入导出分享、试玩并记录成绩。

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 类型检查
npm run check

# 运行单元测试
npm test
```

## 游戏规则

### 1. 游戏目标

在 50 回合内达到 100 分获胜。

### 2. 基本操作

#### 移动方式

- **鼠标点击**：点击玩家相邻的格子进行移动
- **键盘控制**：
  - `W` / `↑` 向上移动
  - `S` / `↓` 向下移动
  - `A` / `←` 向左移动
  - `D` / `→` 向右移动
- **快捷键**：
  - `Ctrl + Z` 撤销上一回合
  - `Ctrl + N` 开始新游戏

### 3. 游戏元素

| 元素 | 图标 | 说明 |
|--------|------|------|
| 巡逻单位 | ● 绿色发光圆点 | 玩家控制的单位，从左上角 (0,0) 开始 |
| 普通事件 | ◆ 橙色 | 捕获后 +10 分 |
| 奖励事件 | ★ 金色 | 捕获后 +30 分 |
| 危险事件 | ☠ 红色 | 捕获后 -20 分，**游戏立即结束** |
| 障碍物 | ▓ 灰色方块 | 不可通过 |

### 4. 移动规则

- 玩家每回合只能向**上下左右**四个方向移动 **1 格
- **不能移动到棋盘外**（8×8 边界）
- **不能穿过障碍物**
- **非法移动时**：
  - 系统提示"非法移动"
  - 回合数**不变化**
  - 分数**不变化**
  - 事件位置**不变化**
  - 游戏状态完全不变

### 5. 事件系统

#### 事件捕获

- 玩家移动到事件所在格子即捕获该事件
- 事件捕获后立即从棋盘上消失
- 分数立即结算

#### 系统回合

每次玩家合法移动后，系统会执行以下操作之一：

- **50% 概率**：随机移动一个现有事件 1 格
- **30% 概率**：生成一个新事件
  - 70% 普通事件 (+10分)
  - 15% 奖励事件 (+30分)
  - 15% 危险事件 (-20分)
- **20% 概率**：无变化

### 6. 失败条件

满足以下任一条件，游戏立即结束：

1. **捕获危险事件**：移动到红色危险事件格子
2. **分数为负**：分数低于 0 分
3. **回合耗尽**：50 回合结束时分数未达到 100 分

### 7. 胜利条件

- 分数达到或超过 100 分，游戏胜利

### 8. 撤销功能

- **撤销上一回合**：恢复到上一回合开始前的状态
- 撤销后恢复：
  - 玩家位置
  - 所有事件位置
  - 分数
  - 回合数
  - 回放日志
- 可多次撤销，直到游戏初始状态
- 回放模式下无法撤销

### 9. 存档系统

#### 手动存档

- 5 个存档槽位
- 可自定义存档名称
- 存档内容包括：
  - 当前棋盘状态
  - 玩家位置
  - 所有事件位置
  - 分数和回合数
  - 完整历史记录
  - 回放日志

#### 自动存档

- 每次合法移动后自动保存
- 关闭页面后重新打开可读取自动存档
- 自动存档独立于手动存档

#### 读取存档

- 读取存档后可继续游戏
- 读取存档后可进行回放

### 10. 回放功能

- 查看完整的游戏过程
- 支持：
  - 逐帧前进/后退
  - 自动播放（0.5x / 1x / 2x / 4x 速度)
  - 拖动进度条跳转到指定回合
  - 退出回放回到当前状态

### 11. 操作日志

记录每回合的详细操作：

- 移动记录（方向、起始位置、目标位置)
- 事件捕获（事件类型、分数变化)
- 系统操作（事件移动、新事件生成)
- 非法移动提示
- 游戏结束原因

### 关卡工坊档案馆模块

工坊档案馆支持玩家将自制关卡存为草稿、发布到独立列表、编写标题备注和版本说明，从档案馆直接进入试玩，并带着成绩回到原草稿继续修改。

#### /workshop 入口

- **主页入口**：主页标题栏右侧「关卡工坊」按钮，点击跳转到 `/workshop`
- **工坊返回**：工坊页面左上角「返回主页」按钮，点击回到 `/`
- **工坊中返回继续编辑**：主页处于工坊模式时，标题栏右侧「回到工坊继续编辑」按钮

#### 档案馆操作顺序

1. **创建草稿**：点击「+ 新建关卡」→ 编辑器中布置元素 → 填写名称和描述 → 「保存关卡」→ 自动存入草稿列表
2. **发布关卡**：草稿关卡卡片上点击「发布」按钮（✓）→ 填写版本说明 → 确认发布 → 状态变为「已发布」，版本号自动递增
3. **试玩关卡**：点击任意关卡卡片上的「试玩」按钮（▶）→ 跳转主页开始试玩 → 游戏结束后成绩自动记录到该关卡
4. **返回继续编辑**：试玩中途或结束后点击「回到工坊继续编辑」→ 自动弹出编辑器，加载试玩前的快照
5. **撤回发布**：已发布关卡卡片上点击「撤回发布」按钮（⚠）→ 状态恢复为「草稿」
6. **编辑修改**：点击「编辑」按钮（✎）→ 修改后保存 → 已发布关卡自动变回草稿状态
7. **导出/导入**：单个关卡点击「导出 JSON」，全部关卡点击「导出全部」；点击「导入关卡」选择 JSON 文件
8. **冲突处理**：导入同名关卡时弹出选择器，可选择「覆盖」或「另存为新关卡」
9. **删除关卡**：点击「删除」按钮 → 二次确认后删除（含所有成绩记录）

#### 状态与日志区分

| 状态 | 标签颜色 | 日志标记 | 说明 |
|------|----------|----------|------|
| 官方局 | 青色 | `official` | 默认游戏模式 |
| 草稿局 | 黄色 | `workshop-draft` | 未发布的工坊关卡 |
| 已发布局 | 绿色 | `workshop-published` | 已发布的工坊关卡 |

### 关卡工坊验收步骤

> 以下步骤覆盖全部工坊需求（创建/编辑/删除、发布/撤回、导入冲突、跨重启恢复、试玩后回工坊继续修改），可按顺序手动走一遍。

#### A. 入口与基础导航

A1. 打开 `http://localhost:5173/` → 主页标题栏右侧应能看到 **「关卡工坊」** 按钮  
A2. 点击「关卡工坊」→ URL 跳转到 `/workshop`，进入工坊页面  
A3. 工坊页面左上角有 **「返回主页」** 按钮，点击回到 `/`  
A4. 主页点击「开始新游戏」→ 新的官方局，日志面板顶部标签为「官方局」青色

#### B. 创建、编辑、保存自定义关卡

B1. 进入工坊 → 点击顶部 **「+ 新建关卡」** → 弹出编辑器  
B2. 编辑器顶部有 6 个工具按钮（橡皮擦、玩家、障碍、普通、奖励、危险），默认选中「玩家」  
B3. 放置元素：
  - 点击 (4,4) 附近 → 玩家移动到该格，左下角实时统计显示玩家坐标同步变化
  - 切换到「障碍」工具 → 在 (2,2)、(3,3)、(5,5) 点击三次 → 统计栏障碍数显示 `3`
  - 切换到「普通」→ 在 (1,0)、(0,1) 各点一个 → 普通事件数显示 `2`
  - 切换到「奖励」→ 在 (7,7) 点一个 → 奖励事件数显示 `1`
  - 切换到「危险」→ 在 (6,6) 点一个 → 危险事件数显示 `1`
  - 切换到「橡皮擦」→ 点击 (6,6) → 危险事件数回到 `0`
B4. 关卡命名：顶部名称框输入 `验收关卡1`，描述输入 `基础验收用`  
B5. 点击 **「保存并试玩」** → 弹出校验提示「保存成功」→ 自动跳转主页并开始工坊试玩  
B6. 主页检查：
  - 顶部出现紫色 **「工坊模式：验收关卡1」** 标签
  - 出现 **「返回工坊继续编辑」** 和 **「退出工坊」** 按钮
  - 状态栏顶部紫色条显示关卡名称
  - 日志面板顶部标签为「工坊局」紫色，首条日志内容含「工坊关卡『验收关卡1』开始」
  - 每条移动/捕获日志右上角都有紫色「工坊」小标签，hover 显示关卡名

#### C. 试玩、撤销、存档、回放照常工作

C1. 在工坊关卡中用方向键/WASD 或点击移动若干步 → 分数、回合数正常变化  
C2. 按 `Ctrl+Z` 撤销 → 玩家位置、分数、回合数正常回退，撤销功能无异常  
C3. 点击「存档」→ 保存到槽位 1 → 然后移动几步 → 再读取槽位 1 → 状态正确恢复  
C4. 点击「回放」→ 进入回放模式 → 逐帧前进后退正常 → 退出回放回到当前状态  
C5. 关闭浏览器标签页 → 重新打开页面 → 自动弹出「发现未完成的工坊局」→ 点继续加载 → 之前的工坊状态完整恢复（含关卡名称、位置、分数、日志）

#### D. 返回工坊继续编辑、编辑已有关卡、删除关卡

D1. 试玩中途（或结束后）点击 **「返回工坊继续编辑」** → 跳回工坊页面并自动弹出编辑器，加载之前的快照（玩家位置、事件、障碍全部保持试玩前的状态）  
D2. 在编辑器中修改：将玩家移到 (0,0)，再加两个障碍 → 点击 **「保存」**（注意是保存不试玩）→ 提示成功  
D3. 返回工坊列表 → 「验收关卡1」卡片统计数字已更新（障碍数 +2）  
D4. 点击该卡片 **「编辑」** → 编辑器加载最新保存内容 → 确认玩家和障碍是 D2 中的修改  
D5. 关闭编辑器 → 点击该卡片 **「删除」** → 弹出二次确认 → 确认后卡片从列表消失，统计栏总关卡数 -1  
D6. 重新新建并保存一个简单关卡 `验收关卡2` 备用

#### E. 导出 / 导入 JSON、同名冲突处理

E1. 在工坊列表点击「验收关卡2」卡片上的 **「导出 JSON」** → 浏览器下载一个 `.json` 文件  
E2. 用文本编辑器打开该文件 → 修改 `name` 字段值为 `导入关卡A`，保存  
E3. 点击工坊顶部 **「导入关卡」** → 选择修改后的文件 → 提示「1 个关卡已导入」→ 列表新增「导入关卡A」  
E4. 再次导入同一个文件（关卡名叫「导入关卡A」，和 E3 中的同名） → 弹出冲突选择器：
  - 显示「导入关卡A」与现有同名
  - 选择 **「另存为」** → 导入后列表出现「导入关卡A (1)」
E5. 再一次导入同一文件 → 冲突时选择 **「覆盖」** → 列表中「导入关卡A」数量不变，但 obstacle 统计和导入文件一致（覆盖成功）
E6. 用记事本创建一个无效 JSON（例如写一段乱码）→ 尝试导入 → 提示「导入失败，格式错误」，现有列表未变化（污染保护生效）
E7. 点击工坊顶部 **「导出全部」** → 下载批量导出包，再次新建浏览器配置清空 localStorage 后 → 导入该批量包 → 全部关卡和成绩恢复

#### F. 跨重启恢复（localStorage 持久化）

F1. 清空 localStorage 后，重新打开页面：
  - 工坊列表为空
  - 无「继续编辑」提示弹窗
  - 成绩统计全为 0
F2. 创建 2 个关卡并保存，每个都试玩一局（不管胜负），其中一个关卡编辑到一半（编辑器有修改未保存）  
F3. **不保存编辑器直接关闭浏览器** → 重新打开页面 → 工坊列表 2 个关卡存在，成绩都已记录  
F4. 打开页面时若上次有未完成的编辑 → 弹窗提示「发现未保存的编辑快照，是否继续」：
  - 点「继续编辑」→ 自动打开编辑器，内容为关闭前最后快照
  - 或点「忽略」→ 清除 lastEditing 记录

#### G. 日志与存档中的来源标记

G1. 开启官方局 → 存档到槽位 2 → 开工坊局 → 存档到槽位 3  
G2. 使用「导出存档包」功能导出 → 解压/打开 JSON → 每条存档记录应包含：
  - 官方局存档：`gameState.levelSource === 'official'`，无 `levelId`/`levelName`
  - 工坊局存档：`gameState.levelSource === 'workshop'`，包含 `levelId` 和 `levelName`
G3. 工坊局日志中至少包含 3 条以上 `levelSource: 'workshop'` 的记录，官方局对应为 `official`

#### H. 单元测试 & 构建验证

H1. 运行 `npm run check` → 类型检查无错误  
H2. 运行 `npm test` → 全部 50+ 用例通过（含 31 条工坊专项用例）  
H3. 运行 `npm run build` → 构建成功，无 TS 编译错误
